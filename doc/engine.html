class Board(width: Int, height: Int) extends TriGrid[TriHolder]

An instance of the class Board represents a two-dimensional hexagon that can be inhabited by triangle tiles. This kind of "Board" is a TriGrid whose elements are TriHolder objects.

An instance of the class Board represents a two-dimensional hexagon that can be inhabited by triangle tiles. This kind of "Board" is a TriGrid whose elements are TriHolder objects.

Triangle tile — TriTile objects — can be added to the board, and the board object maintains a listing of the added tiles.

In the scope of this project, the instances of the Board can only have width=7 and height=4. The constraints come from the constant hashmap that translate between GridPos and TriGridPos. This issue must be improved with better implementation rather than a constant hashmap if we want a scalable project.

class GameBoard extends Board

A GameBoard is an extension of the Board object. This means that a GameBoard is also a board that is inhabited by triangle tiles.

A GameBoard is an extension of the Board object. This means that a GameBoard is also a board that is inhabited by triangle tiles.

Besides the basic functionality of holding the TriHolder and the TriTile objects, a GameBoard also contains some important methods that implement the logic of the game.

For example, generateSolution() to generate a valid solution of a game, and shuffleTiles() to shuffle the tiles and start the game.

In the scope of this project, the instances of the GameBoard can only have width=7 and height=4. This constraint is explained well in Board and engine.grid.TriGrid.

final case class TriTile(var _a: Int, var _b: Int, var _c: Int)

The class TriTile represents the triangle tile that can be added to, removed from, and rotated around in a hexagon.

The class TriTile represents the triangle tile that can be added to, removed from, and rotated around in a hexagon.

Each TriTile can be added to a TriHolder in either a GameBoard or a WaitingBoard.

Each TriTile has the following properties:

  • var a, b, c: coordinate in triangular system
  • var p1, p2, p3: three corners of the triangle, always start from the rightmost corner and follow clockwise order
  • var e1, e2, e3: three edges correspond to corner pairs (p1-p2), (p2-p3), (p3-p1)

The class TriTile is mutable, i.e., the coordinates a, b, c can be changed. These changes can be done with class's method updateCoords().

In addition, TriTile also has values attached to the edges of the triangle. When first creating a TriTile instance, the edges all have value -1. These values can be changed with method updateEdgeValues().

class WaitingBoard extends Board

A WaitingBoard is an extension of the Board object. This means that a WaitingBoard is also a board that is inhabited by triangle tiles.

A WaitingBoard is an extension of the Board object. This means that a WaitingBoard is also a board that is inhabited by triangle tiles.

A WaitingBoard is simply a containers for the TriHolder and TriTile objects. It serves as a "free-to-use" zone with the main purpose of supporting the players. It is where the players can put some tiles on to simplify their game.

In the scope of this project, the instances of the WaitingBoard can only have width=7 and height=4. This constraint is explained well in Board and engine.grid.TriGrid.
